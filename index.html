Responsive Sticky Footer Tutorial

The following method for creating footers has the following benefits over most others found online currently:

  1. This footer is fully responsive and looks good on any size device no matter the amount of content on the page.
  2. No positioning required.
  3. No media queries required.
  4. Easy to follow and understand.
  5. Fantastic browser support.

  2. There is a footer element with the class attribute of footer. This .footer will represent the footer of the web page.

Things to note:

  1. The html, and body elements are given a height of 100% and their default margin and padding is removed.

    • By default, elements only stretch to fit their children. Setting the height of the html and body ensures that the web page is always at least as tall as the viewport.
    • Removing the margin and padding helps keep the footer at the bottom of the page. Without any intervention, most browsers will add some default margin and padding to the body and html elements.

  2. The #wrapper element has a min-height property set to 80%, and a padding-bottom property set to 10px.

    • Giving #wrapper a minimum height of 80% ensures that even if there is no content in #wrapper, it will be at least 80% of the viewport/browser screen's height. The other 20% will be the min-height of .footer. You can adjust these numbers so long as they both add up to 100%.
    • The padding-bottom property may not be necessary in every case and its value can vary. This is the added distance between the bottom of the content in #wrapper and the top of .footer. By default, the content in #wrapper will never spill inside of .footer, but you can add to the padding-bottom if you need more room.

  3. The .footer element has a min-height property set to 20%, an overflow property set to auto, and a background property set.

    • As mentioned in the last step, the min-height of #wrapper and .footer must add up to 100% in order to display properly when there is little or no content in #wrapper. Both #wrapper and .footer will stretch to fit any amount of content inside of them. This is part of what makes this footer method responsive.

    • The overflow: auto rule inside of .footer helps keep the .footer at the bottom of the page even when there is a variable size of child content. This is technically not required when using reset.css.
